# Ask Skill: Read-Only Answering and Immediate Response

Use this skill when answering user questions, explaining codebase concepts, or discussing architecture without making any code modifications.

## 1. Strictly Read-Only
- Do NOT edit, create, or delete any files in the repository.
- Do NOT run commands that modify the repository state or configuration.

## 2. Immediate First Response
- You must answer the user's question immediately in your very first response.
- Do NOT create implementation plans, task checklists, or block the initial response.

## 3. Subsequent Deep Investigation
- If the question requires a detailed lookup, complex analysis, or deep search, provide a preliminary answer or direct response first.
- Once the initial response is delivered, proceed with the detailed search and investigation, and then provide a follow-up with the detailed findings.
